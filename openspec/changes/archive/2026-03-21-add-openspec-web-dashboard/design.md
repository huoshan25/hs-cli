## Context

本次目标是把 OpenSpec 的核心浏览与操作体验迁移到 Web 面板，保持与原有 CLI/TUI 数据语义一致，同时提供更高信息密度和可视化能力。

## Goals / Non-Goals

**Goals:**
- 在不改变 OpenSpec 数据模型的前提下提供 Web 交互层
- 保持与原有工作流一致：浏览、检索、切换项目、执行变更动作
- 支持快捷键和主题，满足日常高频使用

**Non-Goals:**
- 不重构 OpenSpec 核心数据结构
- 不引入与 OpenSpec 无关的业务能力
- 不在规格中记录纯样式微调或代码重排

## Decisions

**Web 面板作为 `openspec` 的可选 UI 模式**

通过 `--ui web` 明确启用，避免影响已有 CLI/TUI 使用习惯。

**信息架构与现有域模型对齐**

采用 `overview/specs/changes/archive` 四标签，减少认知切换成本。

**动作执行闭环内置在变更详情**

“严格验证/确认归档”直接在变更详情触发并回显结果，避免上下文跳转。

**键盘优先交互**

保留并扩展快捷键（导航、搜索、项目切换、弹层关闭），提升高频操作效率。

## Risks / Trade-offs

- [风险] Web 与 CLI/TUI 行为漂移
  - 通过共享语义约束与统一规格场景降低偏差
- [风险] 复杂 Markdown 渲染稳定性
  - 采用可回退渲染策略，失败时仍保证可读
- [取舍] 增加前端构建产物与分发链路
  - 换取更强的可视化表达与可扩展 UI 能力
